Como usamos pytorch el entrenamiento fue de esta manera:

El entrenamiento en PyTorch fue un ciclo de "prueba y error":

Forward: La red lanza una predicción inicial.

Loss (BCELoss): Comparamos esa predicción con el resultado real para ver qué tan equivocado estuvo el modelo.

Backpropagation: El sistema viaja hacia atrás por la red para encontrar qué conexiones fallaron.

Optimizer (Adam): Ajustamos los "pesos" de la red para que en la siguiente vuelta el error sea menor.
